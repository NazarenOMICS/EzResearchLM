# Rules Of Writing

Use this file when drafting academic prose from NotebookLM QA outputs.

## Core Rule

Do not add claims that are not supported by cited evidence in the available QA
or source summaries.

## Style

- Prefer clear academic prose over bullet lists unless the user requests a table
  or checklist.
- Separate reported evidence from interpretation.
- State uncertainty explicitly.
- Avoid inflated language.
- Avoid claims about mechanism, magnitude, causality, or generalizability unless
  the cited source supports them.
- Do not use em dashes.

## Citation Discipline

- Preserve NotebookLM citation markers when drafting from QA outputs.
- If citations do not support a sentence, remove the sentence or mark the gap.
- If evidence is incomplete, say so directly.

## Thesis-Ready Output

A paragraph is thesis-ready only when:

- it answers the question asked;
- every strong claim has source support;
- population, organism, method, dose, time, and context are not mixed across
  sources without saying that this is an inference;
- limitations are stated where evidence is indirect.
